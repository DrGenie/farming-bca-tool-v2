User guide

This guide explains what each tab does, how the calculations work, how the uploaded workbook is interpreted, and how to read the outputs. It is written so that a farmer, adviser, project officer, or researcher can use the tool without needing a separate briefing.

1. What the tool does

The tool compares treatments from an agricultural trial workbook. Each replicate row supplies one treatment observation. The tool reads crop yield and the individual direct cost components, calculates a direct cost total for each replicate, averages the replicate rows within each treatment, and then applies the user’s grain price and discounting settings to produce present value measures.

The current version is intentionally focused on direct market effects only. Benefits are valued from crop yield. Costs are valued from the cost component fields in the workbook. Indirect or environmental co-benefits are not included in this version.

2. How each tab works

Introduction gives the background, scope, acknowledgements, and main workflow.

Data is where the workbook is uploaded, checked, summarised, and previewed. This tab shows whether the workbook was read correctly and whether a control treatment was found.

Settings is where the grain price, analysis period, and discount schedule are set. These choices affect all present value results.

Results is where the analysis is run. It shows headline metrics, a comparison table, and the full treatment ranking table. It also provides prepared prompts for ChatGPT and Copilot.

Sensitivity allows direct numeric testing of alternative discount rates, grain prices, benefits, and costs.

Help provides quick answers and opens this guide in a separate window.

3. How the workbook is read

The workbook should use the official template and contain a sheet named Trial_Data. Each row below the header should represent one replicate. The tool uses the following fields directly: Treatment ID, Replicate ID, Amendment Name, Practice Change Label, Crop_yield_t_ha, and all cost component fields.

The control is identified from the control row in the workbook. In the supplied trial data, the control is the row where the amendment name is Control and the practice label is No change.

4. Formulas used in the tool

Calculated direct cost per replicate is the sum of the main direct cost components in that row.

Calculated direct cost = Seed cost + Total labour + Total machinery + Fertiliser + Pesticides + Fuel + Marketing + Repairs and maintenance + Support services + Electricity + Water + Insurance services + Overheads + Interest paid on loans + Structure maintenance + Other costs

Average yield for a treatment is the arithmetic mean of the replicate crop yield values for that treatment.

is the arithmetic mean of the replicate crop yield values for that treatment.

Average direct cost for a treatment is the arithmetic mean of the calculated direct cost totals across that treatment’s replicate rows.

Annual benefit per hectare is average yield multiplied by grain price.

Annual benefit = Average yield × Grain price

Present value factor is the sum of discount factors across the selected analysis years.

PV factor = Σ 1 / Π(1 + r_t)

Present value of benefits is annual benefit multiplied by the present value factor.

Present value of costs is average direct cost multiplied by the present value factor.

Net present value equals present value of benefits minus present value of costs.

Benefit-cost ratio equals present value of benefits divided by present value of costs.

Gross profit margin is calculated as ((PV benefits − PV costs) / PV benefits) × 100.

Difference versus control is the treatment net present value minus the control net present value.

5. Detailed numerical example from the included dataset

This worked example uses the included uploaded trial data, a grain price of $500 per tonne, an analysis period of 10 years, and a constant annual discount rate of 5.0%.

The present value factor for 10 years at 5.0% is approximately 7.7217. This is obtained from adding the yearly discount factors from year 1 to year 10.

For the Gypsum treatment, the average crop yield across the replicate rows is 3.3325 t/ha. The annual benefit is therefore 3.3325 × $500 = $1666.25 per hectare.

The calculated direct cost for each replicate is obtained by summing the main direct cost components in that row. For Gypsum, this means seed cost, total labour, total machinery, fertiliser, pesticides, fuel, marketing, repairs and maintenance, support services, electricity, water, insurance services, overheads, interest paid on loans, structure maintenance, and other costs. The average calculated direct cost across the Gypsum replicate rows is $812.08 per hectare.

The present value of benefits for Gypsum is therefore $1666.25 × 7.7217 = $12866.34 per hectare.

The present value of costs for Gypsum is therefore $812.08 × 7.7217 = $6270.65 per hectare.

The net present value for Gypsum is $12866.34 − $6270.65 = $6595.69 per hectare.

The benefit-cost ratio for Gypsum is $12866.34 / $6270.65 = 2.05.

The gross profit margin for Gypsum is (($12866.34 − $6270.65) / $12866.34) × 100 = 51.3%.

For the Control treatment, the average crop yield is 2.7525 t/ha and the average calculated direct cost is $846.00 per hectare. The control net present value under the same settings is $4094.41 per hectare.

The difference versus control for Gypsum is therefore $6595.69 − $4094.41 = $2501.28 per hectare.

This means that, under these particular assumptions, Gypsum generates a higher discounted return per hectare than the control.

6. How to interpret the outputs

A positive net present value means discounted benefits exceed discounted costs. A higher net present value indicates a stronger return under the chosen settings.

A benefit-cost ratio above 1.00 means benefits exceed costs. Ratios further above 1.00 indicate a larger benefit return for each discounted dollar of cost.

A higher gross profit margin means a larger share of discounted benefits remains after discounted costs are deducted.

The difference versus control helps answer the practical question of whether a treatment performs better or worse than the reference practice.

7. Common issues and what they mean

If the workbook cannot be read, save the Excel file and upload it again using the official template.

If no results appear, check that the workbook contains data rows, that yield values are present, and that the control treatment can be identified.

If the validation note says the reported total farm cost differs from the summed cost components, the analysis will still run. The tool uses the summed cost components because that is the method adopted for this version.
